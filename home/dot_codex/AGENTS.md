## 使用言語

- ユーザーとの全てのやり取りを日本語で行うこと
  - 会話、進捗報告、質問、成果物の説明を含む

## 日本語表現のルール

### 文体のルール

#### 文（センテンス）

- 文は「です・ます調」で統一すること

#### 箇条書き

- 箇条書きの項目は、規約文・仕様書・箇条書きで使われる簡潔な言い切り表現を使用すること
  - ただし、説明を要する項目は「です・ます調」の文で記述して問題ない
- 複数の文（センテンス）を使用しないこと
  - 文（センテンス）が長くなる場合はインデントを下げ項目を分けて記述する
- 句点（。）を使用しないこと
- 意味の切れ目に必要な場合、読点（、）は使用してよい

### 特殊なルール

#### 全角文字と半角文字が隣接する場合のルール

- 日本語（全角）と半角文字が混在しても読みやすくなるよう、両者が隣接する境界には半角スペースを 1 つ追加すること
  - 例: 「彼は Word を使う」「第 3 章」「過去 13 か月」
- ただし、記号や約物はそれ自体が区切りとして働くため、前後に半角スペースを追加する必要はない
  - 句読点、括弧、引用符、スラッシュなど
    - 例: 「対応形式は PNG、JPEG。」「（Word 形式）」「オン/オフ」「50%」
- URL、メールアドレス、ファイルパス、コード（ブロック・インライン）の内部は対象外とする
  - これらの項目の内部は改変せず原文のまま出力すること
- 判断に迷う場合は、「読みやすさのために半角スペースを追加する」という原則に沿って処理すること

## Git の操作

### git worktree の操作

- gtr コマンドがインストールされている場合は、`git gtr` コマンドを使用してワークツリーを操作すること
  - `git-gtr -v` でインストールされているか確認可能

```bash
git gtr list                  # ワークツリー一覧の表示
git gtr new my-feature        # ワークツリーの作成
git gtr rm my-feature         # ワークツリーの削除
git gtr clean --merged --yes  # ワークツリーのクリーンアップ
```
